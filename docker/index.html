Docker and Docker Compose

Install Docker#

  1. Install prerequisites packets

    sudo apt-get install apt-transport-https ca-certificates curl software-proerties-common

  2. Add docker repository's GPG key

    curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

  3. Add docker repository

    sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"

  4. Refresh Ubuntu packages list

    sudo apt-get update

  1. Install docker

    sudo apt-get install docker-ce

    check the installed version

    docker --version

Install Docker Compose#

  1. Install latest version docker-compose

    sudo curl -L https://github.com/docker/compose/releases/download/1.23.2/docker-compose-`uname -s`-`uname -m` -o /usr/local/bin/docker-compose

    Replace 1.23.2 with currently available version.

  2. Provide execute permission to to docker-compose

    sudo chmod +x /usr/local/bin/docker-compose

    check version number

    docker-compose --version

Add Linux User to Docker Group#

Running and managing docker containers requires sudo privileges.

sudo usermod -aG docker ${USER}

Basic Docker and Docker Compose#

Docker folder and permission#

Create folder as storage docker file

mkdir ~/docker

Setup apporopriate permission to docker folder

sudo setfacl -Rdm g:docker:rwx ~/docker
sudo chmod -R 775 ~/docker

create docker-compose.yml file

nano ~/docker/docker-compose.yml

Managing docker#

  1. Starting docker

    docker-compose -f ~/docker/docker-compose.yml up -d

  2. See docker containers

    docker ps -a

  3. Check docker containers logs

    docker-compose logs

    check spesific container's logs

    docker-compose logs [container-name]

  4. Stop containers

    docker-compose -f ~/docker/docker-compose.yml down

    stop single container

    docker-compose stop [container-name]

  5. Docker cleanup

    docker system prune
    docker image prune
    docker volume prune
